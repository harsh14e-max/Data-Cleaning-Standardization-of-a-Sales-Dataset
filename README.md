# Data-Cleaning-Standardization-of-a-Sales-Dataset

## Project Overview

Raw data is often messy, inconsistent, and not ready for analysis. In this project, I transformed a poorly structured sales dataset into a clean, standardized, and analysis-ready dataset using Microsoft Excel.

The original dataset contained missing values, inconsistent formats, mixed data types, and non-standardized categories, making it unreliable for reporting and business analysis.

The objective of this project was to improve data quality and create a structured dataset that could be used confidently for analysis and decision-making.

---

## Before vs After

### Before Cleaning

The raw dataset contained several common data quality issues:

* Inconsistent column names
* Missing values (null, N/A, ?, blanks)
* Mixed text and numeric values
* Inconsistent date formats
* Random symbols and formatting issues
* Non-standardized categorical values
* Potential duplicate records

### After Cleaning

The dataset was transformed into a structured and reliable format with:

* Clear and meaningful column names
* Standardized data types
* Clean numerical fields
* Consistent categorical values
* Improved data quality
* Analysis-ready structure

---

## Data Cleaning Process

### 1. Understanding and Renaming Columns

I reviewed the dataset column by column to understand the purpose of each field and renamed columns to make them clearer and more meaningful for analysis.

### 2. Handling Missing and Invalid Values

I identified missing and invalid entries such as:

* null
* N/A
* ?
* Blank cells

Instead of deleting rows unnecessarily, I handled these values logically to preserve as much useful information as possible.

### 3. Fixing Data Types

Several columns contained inconsistent formats. I standardized them by:

* Converting dates into proper date format
* Converting quantities into numeric values
* Converting prices into numeric values
* Formatting sales amounts as decimals

### 4. Standardizing Numerical Data

Numeric fields contained text, symbols, and inconsistent formatting. I cleaned these fields by removing unnecessary characters and ensuring values could be used reliably in calculations.

### 5. Cleaning Categorical Data

Categorical fields such as:

* Payment Mode
* Sales Channel
* Region
* City

were standardized by correcting inconsistent spellings, formatting, and capitalization to ensure uniform categories throughout the dataset.

### 6. Data Validation

Finally, I reviewed the cleaned dataset for consistency, checked for duplicate records, and validated the data to ensure it was ready for analysis and reporting.
